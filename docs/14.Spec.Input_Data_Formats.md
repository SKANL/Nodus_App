# 14. Spec: Registration & Rubric Standards

> **Status:** Active
> **Purpose:** Detailed requirements for Data Input (Self-Service) and Config.

## 1. Student Registration Input (The Form)

Students register themselves. The System (Server) creates the record.

### Required Fields (Student)

| Field              | Type     | Min/Max Rules     | Description                 |
| :----------------- | :------- | :---------------- | :-------------------------- |
| **`Project Name`** | Text     | 3-64 chars        | The official title.         |
| **`Category`**     | Dropdown | One of Config key | E.g., "Software", "Social". |

### Optional Fields (Student)

| Field             | Type | Rules         | Description            |
| :---------------- | :--- | :------------ | :--------------------- |
| **`Members`**     | Text | Max 128 chars | Names of team members. |
| **`Github Link`** | URL  | Valid HTTPS   | Link to source code.   |
| **`Description`** | Text | Max 140 chars | Twitter-style pitch.   |

### System Generated (ReadOnly)

- **`Project ID`:** Sequential (`P-001`, `P-002`) or Hash.
- **`QR Payload`:** `nodus://vote?pid=P-001&cat=Soft`

## 2. Input: Rubric Configuration (The Scoring Rules)

Admin defines "what to judge" via JSON or UI.

### Structure (Stored in `Event.RubricJson`)

```json
{
  "criteria": [
    {
      "id": "c1",
      "label": "Originality",
      "description": "How novel is the idea?",
      "min": 1,
      "max": 10,
      "weight": 1.0,
      "step": 1
    },
    {
      "id": "c2",
      "label": "Execution",
      "description": "Does it crash?",
      "min": 0,
      "max": 5,
      "weight": 2.0,
      "step": 0.5
    }
  ]
}
```
