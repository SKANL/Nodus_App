# 13. UX: Admin Dashboard (The Control Tower)

> **Status:** Draft
> **Platform:** Windows Desktop (Mauai / WinUI)
> **Role:** The Central Nervous System

## 1. Pre-Event Setup Flow (At Home/Office)

The Admin must configure the event _before_ going offline.

### A. Event & Registration Setup

1.  **Create Event:** Admin types "Hackathon 2026" and sets Categories (Software, Hardware).
2.  **Generate Registration QR:** Admin clicks "Enable Registration".
    - A QR Code appears: "Scan to Register Project".
    - Admin projects this on the wall.
3.  **Rubric Config:** Admin defines the scoring criteria (e.g., "Innovation (1-10)").

### B. Judge Access Setup (The "Authentication" QR)

1.  **Action:** Admin clicks "Generate JUDGE QR" (Blue Button).
2.  **Config:** "Set Event Password" (Required, e.g., `Tesla2026`).
    - _Security:_ Encrypts the QR payload.
3.  **Output:** A distinct QR code (Blue Border) for Judges to scan.

### C. Security Generation

1.  **Action:** Click "Generate Event Keys".
2.  **Result:** System creates `GlobalSalt` + `SharedAESKey` + `ServerUUID`.
3.  **Output:** Display the **MASTER QR CODE** on screen (to be projected or printed).

During the event, this is the "Flight Control" screen.

### A. Live Registration Monitor

- **View:** Real-time list of registering students.
- **Action:** Approve/Reject/Edit.
